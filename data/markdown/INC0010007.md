# INC0010007

**Short Description:** Podman containers on RHEL 8 unable to pull images from private registry
**Priority:** 4
**State:** 6

## Description
Developers on RHEL 8 workstations and dev servers report that 'podman pull' commands fail with 'x509: certificate signed by unknown authority' when pulling from the internal container registry registry.internal.company.com:5000.

## Resolution
ROOT CAUSE ANALYSIS:
The internal container registry SSL certificate was renewed 3 days ago. The new certificate was signed by an updated internal CA that had not been distributed to RHEL 8 hosts. Podman uses the system certificate trust store, which did not contain the new CA certificate.

RESOLUTION STEPS TAKEN:
1. Reproduced the error: podman pull registry.internal.company.com:5000/myimage:latest
   Output: Error: x509: certificate signed by unknown authority.
2. Retrieved the new internal CA certificate from the PKI team:
   company-internal-ca-v2.crt.
3. Copied the CA cert to the system trust anchor directory on affected hosts:
   cp company-internal-ca-v2.crt /etc/pki/ca-trust/source/anchors/.
4. Updated the system-wide CA trust store:
   update-ca-trust extract.
5. Verified the cert was added: trust list | grep 'Company Internal CA'.
6. Re-tested: podman pull registry.internal.company.com:5000/myimage:latest —    pull succeeded.
7. Rolled out the fix to all 35 affected RHEL 8 hosts using an Ansible ad-hoc    playbook targeting the 'rhel8-dev' inventory group.
8. Confirmed resolution across all hosts by running a post-task smoke test.
9. Notified all development teams that image pulls were restored.
10. Raised a change request to include new CA distribution in the standard     certificate renewal runbook and Satellite content view.

PREVENTIVE MEASURES:
Integrated CA certificate deployment into the PKI renewal pipeline via Ansible Tower so hosts are updated automatically on next renewal.

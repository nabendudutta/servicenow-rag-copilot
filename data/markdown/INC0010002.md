# INC0010002

**Short Description:** RHEL 9 system fails to boot after kernel update
**Priority:** 3
**State:** 7

## Description
Production RHEL 9 server RHEL-PROD-01 failed to boot following a scheduled kernel update from 5.14.0-362 to 5.14.0-427. The system is stuck at the GRUB menu and does not proceed to the login prompt. Server hosts critical middleware services.

## Resolution
ROOT CAUSE ANALYSIS:
The system failed to boot because the new kernel 5.14.0-427 introduced a breaking change with the third-party NVIDIA driver module that was installed on the host.

RESOLUTION STEPS TAKEN:
1. Accessed the GRUB menu during startup by pressing 'e' to edit the boot entry.
2. Selected the previous known-good kernel version 5.14.0-362 from the advanced    boot options menu to restore system access.
3. Logged in successfully and confirmed all services were running on the old kernel.
4. Identified the incompatible kernel module using: lsmod | grep nvidia.
5. Removed the incompatible NVIDIA driver: dnf remove nvidia-driver.
6. Locked the kernel version temporarily: dnf versionlock add kernel-5.14.0-362.
7. Set the default GRUB boot entry to the working kernel:
   grub2-set-default 'Red Hat Enterprise Linux (5.14.0-362) 9.3'.
8. Ran grub2-mkconfig -o /boot/grub2/grub.cfg to regenerate the bootloader config.
9. Rebooted the server and confirmed it booted into the correct kernel.
10. Opened a change request to re-test kernel 5.14.0-427 in a staging environment     before re-applying to production.

PREVENTIVE MEASURES:
Added pre-patch validation playbook in Ansible to check for third-party kernel modules before applying kernel updates.

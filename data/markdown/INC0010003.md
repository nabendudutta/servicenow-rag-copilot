# INC0010003

**Short Description:** Jenkins pipeline failing with 'No space left on device' error
**Priority:** 3
**State:** 7

## Description
Jenkins master node JENKINS-MASTER-01 is reporting 'No space left on device' errors across all running pipelines. All builds are failing at the checkout stage. Development teams cannot merge or deploy code.

## Resolution
ROOT CAUSE ANALYSIS:
The Jenkins workspace directory /var/lib/jenkins/workspace consumed all available disk space (500 GB partition) due to accumulated build artifacts and workspace directories from abandoned pipeline runs over the past 90 days.

RESOLUTION STEPS TAKEN:
1. Confirmed disk usage via: df -h and du -sh /var/lib/jenkins/* to identify the    largest directories consuming space.
2. Identified 230 GB of stale workspace directories from deleted or renamed jobs.
3. Stopped Jenkins service gracefully: systemctl stop jenkins.
4. Manually deleted stale workspace directories:
   rm -rf /var/lib/jenkins/workspace/<stale-job-names>.
5. Cleared the Jenkins build history archive older than 30 days from the UI.
6. Removed old .hpi and .jpi plugin backup files from /var/lib/jenkins/plugins.
7. Freed 280 GB of disk space; partition now at 44% utilization.
8. Restarted Jenkins: systemctl start jenkins and verified all pipelines resumed.
9. Configured Workspace Cleanup Plugin on all existing jobs to wipe workspace    before each build via job DSL update.
10. Set up a cron job to run a disk-space alert script when usage exceeds 75%:
    0 6 * * * /usr/local/bin/disk_alert.sh >> /var/log/disk_alert.log 2>&1.

PREVENTIVE MEASURES:
Implemented Jenkins Configuration as Code (JCasC) policy to enforce workspace cleanup and artifact retention for a maximum of 10 builds.
